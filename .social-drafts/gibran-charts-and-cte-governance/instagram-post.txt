Added charts to my governed analytics engine 📊 — and the most useful thing I learned is that the charts weren't the point.

Quick context: Gibran lets you ask data questions in plain English, with a hard guarantee — it answers from your real schema or says "I don't know." There's no LLM in the SQL path, so it physically can't invent a column name. This update shipped three things:

📈 A real chart layer — table/chart toggle, six chart types, multi-series — lazy-loaded, so the app stays tiny until you actually open a chart.

🏆 A question it couldn't ask before: "biggest 5 customer per country by spend." That's a ROW_NUMBER() window function you'd normally hand-write; now it's one English sentence, compiled to a windowed CTE.

🔒 …and the fix that actually mattered. Building the first two surfaced a quiet bug: the row-level-security filter only worked on flat queries. So every advanced metric — cohort retention, funnels, the new per-group ranking — silently broke for any role with row restrictions.

Here's the subtle part: it failed *closed*. It errored instead of leaking, and the tests only ran unrestricted roles — so nobody noticed it was broken for the people who'd use it most.

The filter was landing on the OUTER query, which reads a CTE — not the source table. The fix injects it into the source scan, BEFORE the ranking runs. So a west-region analyst's "top 3 per region" can only ever rank over west-region rows. Correct AND safe — the ranking never even sees data you're not allowed to.

The lesson I keep relearning: a visible feature is what drags a latent bug in the boring infrastructure into the light. The charts were the demo. The governance fix was the substance.
